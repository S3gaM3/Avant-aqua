# WordPress и WooCommerce для проекта «Авант»

## Настройка с нуля (общая схема)

Нужны: **PHP** (обычно 8.1+), **MySQL или MariaDB**, **веб‑сервер** (Apache/Nginx) или готовый стек на хостинге. Дальше — установка WordPress, плагин WooCommerce, постоянные ссылки, ключи REST API и переменные для Next.js.

### 1. Создайте базу данных

В панели хостинга (phpMyAdmin, ISPmanager и т. п.) создайте пустую БД и пользователя с полными правами на эту БД. Запишите:

- имя хоста БД (часто `localhost` или отдельный хост у облака);
- имя базы, логин, пароль.

### 2. Установите WordPress

**На хостинге:** раздел вроде «Установить WordPress» или загрузите архив с [wordpress.org](https://ru.wordpress.org/download/), распакуйте в корень сайта, откройте сайт в браузере — запустится мастер: язык, название сайта, логин администратора, пароль, префикс таблиц (по умолчанию можно оставить).

**Локально без Docker:** [Local WP](https://localwp.com/), Laragon, XAMPP — создайте новый сайт, мастер сам поднимет БД и выдаст URL вида `http://имя-сайта.local`.

Подключите **HTTPS** на продакшене (Let's Encrypt в панели или у регистратора) — для платежей и для меньшего числа сюрпризов с REST из браузера.

### 3. Постоянные ссылки (обязательно для REST)

В админке: **Настройки → Постоянные ссылки** → выберите вариант **«Название записи»** (не «Обычные») → **Сохранить**. Иначе часть REST‑маршрутов может отвечать 404.

### 4. Установите WooCommerce

**Плагины → Добавить новый** → найдите **WooCommerce** → Установить → Активировать. Пройдите краткий мастер (валюта ₽, адрес магазина и т. д.) — можно заполнить минимально и потом править в **WooCommerce → Настройки**.

### 5. Ключи REST API для фронтенда

1. **WooCommerce → Настройки → Дополнительно → REST API**.
2. **Добавить ключ** → описание (например, «Next.js витрина»).
3. Пользователь — ваш админ или отдельный сервисный пользователь.
4. Права: для только каталога достаточно **Чтение**; если позже будете создавать заказы из Next.js — **Чтение и запись** (осторожнее с секретом).
5. Сохраните **Consumer key** и **Consumer secret** — секрет показывается один раз.

### 6. Подключите Next.js

В корне витрины скопируйте `.env.example` в `.env.local` и заполните:

| Переменная                    | Значение                                                                            |
| ----------------------------- | ----------------------------------------------------------------------------------- |
| `NEXT_PUBLIC_SITE_URL`        | URL фронта, например `http://localhost:3000`                                        |
| `NEXT_PUBLIC_WORDPRESS_URL`   | Базовый URL WordPress **без** завершающего `/`, например `https://shop.example.com` |
| `WOOCOMMERCE_CONSUMER_KEY`    | ключ из шага 5                                                                      |
| `WOOCOMMERCE_CONSUMER_SECRET` | секрет из шага 5                                                                    |

Перезапустите `npm run dev`. Каталог на сайте должен подтянуться с WooCommerce; если переменные не заданы, остаётся демо‑каталог из кода.

Проверка API в браузере (подставьте свой домен и ключи):

`https://ваш-сайт/wp-json/wc/v3/products?consumer_key=...&consumer_secret=...`

Должен вернуться JSON (массив товаров или `[]`).

### 7. Гибридная оплата (по желанию)

Если оплата идёт на стороне WordPress: в WooCommerce откройте страницу **Оформление заказа** на сайте и скопируйте её URL в `.env.local`:

`NEXT_PUBLIC_WC_CHECKOUT_URL=https://ваш-сайт/checkout/`

На WordPress ставится плагин эквайера (например, ЮKassa для WooCommerce) и настраивается по инструкции банка.

### 8. CORS и запросы с localhost

Если фронт на `http://localhost:3000`, а WordPress на другом домене, браузер может блокировать запросы к REST. Варианты:

- разрабатывать с тем же доменом через прокси в Next.js (Route Handler переписывает запросы на WP);
- или плагин типа **Headless Mode** / настройка CORS в WordPress (фильтры в `functions.php` или плагин) — осторожно, не открывайте лишнее на проде.

На продакшене часто выносят API на поддомен `api.` или один домен с разными путями за reverse proxy.

## Где размещать WordPress

**Docker не обязателен.** Выберите любой вариант:

- **Хостинг** (shared, VPS, облако с панелью) — стандартная установка WordPress в один клик или вручную.
- **Локально без Docker** — [Local WP](https://localwp.com/), XAMPP, Laragon, WAMP: поднимаете сайт на `http://…` и в `.env.local` указываете этот URL в `NEXT_PUBLIC_WORDPRESS_URL`.

### Опционально: Docker в репозитории

Если Docker вам удобен:

```bash
docker compose up -d
```

Откройте http://localhost:8080 и завершите установку WordPress.

## Импорт каталога

Файл `data/catalog-seed.csv` — образец колонок для импорта через плагины вроде WP All Import или нативный импорт WooCommerce (CSV товаров). Иерархию категорий задайте в WooCommerce до импорта или через колонку «Категории».

## Оплата (Россия)

Подключите плагин эквайера (например, ЮKassa для WooCommerce). Для headless‑витрины часто используют гибрид: каталог на Next.js, финальный шаг — страница `checkout` WooCommerce; переменная `NEXT_PUBLIC_WC_CHECKOUT_URL` ведёт пользователя на этот URL.

## Безопасность и бэкапы

На продакшене используйте ограничение доступа к `/wp-admin`, актуальные версии PHP и WordPress, плагин резервного копирования по расписанию и HTTPS.
